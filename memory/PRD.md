# WM Finance - Product Requirements Document

## Обзор
WM Finance — веб-приложение для финансового учёта малого бизнеса. Компания занимается продажей и производством теплиц, саун и купелей в Польше. Работа ведётся в польских злотых (PLN) и евро (EUR).

**Брендинг:** Made by Knyazev (отображается в футере на всех страницах)

## Технологический стек
- **Frontend:** React 18, Tailwind CSS, Shadcn/UI, Recharts
- **Backend:** FastAPI, Python 3.11
- **Database:** MongoDB (via Motor)
- **Auth:** JWT tokens + Superadmin (без публичной регистрации)
- **AI:** Claude Sonnet 4.5 (Emergent LLM Key)
- **Scheduler:** APScheduler для автоматических Telegram-сводок
- **Integrations:** Telegram Bot, Adesk API

## API Endpoints

### Auth
- `POST /api/auth/login` — вход (логин или email)
- `POST /api/auth/register` — **ОТКЛЮЧЕНО** (403)

### Admin (только superadmin)
- `GET /api/admin/users` — список пользователей
- `POST /api/admin/users` — создать пользователя
- `PUT /api/admin/users/{id}` — обновить пользователя
- `DELETE /api/admin/users/{id}` — удалить пользователя

### Telegram
- `GET /api/bot/summary` — финансовая сводка
- `POST /api/settings/telegram/test` — тест подключения
- `POST /api/settings/telegram/send-summary` — отправить сводку
- `GET /api/settings/integrations` — настройки интеграций
- `PUT /api/settings/integrations/telegram` — обновить настройки

### Adesk Migration
- `POST /api/adesk/test-connection` — проверка подключения
- `POST /api/adesk/start-migration` — запуск миграции
- `GET /api/adesk/drafts` — черновики миграции
- `PUT /api/adesk/drafts/{id}` — редактирование черновика
- `POST /api/adesk/drafts/bulk-update` — массовое обновление
- `POST /api/adesk/confirm-ready` — импорт готовых
- `GET /api/adesk/export-problems` — экспорт проблемных

## Архитектура планировщика (APScheduler)

```python
# Запускается при старте приложения
scheduler = AsyncIOScheduler()

# Telegram сводки
scheduler.add_job(
    send_scheduled_telegram_summary,
    CronTrigger(hour=9, minute=0),  # Каждый день в 9:00 UTC
    id="telegram_summary_job"
)

# Google Sheets бэкап
scheduler.add_job(
    scheduled_google_sheets_backup,
    CronTrigger(hour=6, minute=0),  # Каждый день в 6:00 UTC
    id="google_sheets_backup_job"
)
```

### Telegram логика:
1. Находит всех пользователей с `telegram_auto_summary=True`
2. Проверяет расписание (daily/weekly/monday/friday)
3. Формирует и отправляет сводку через Telegram Bot API

### Google Sheets бэкап:
- Выгружает ВСЕ данные: операции, контрагенты, счета, категории, направления, проекты, плановые платежи
- Каждая сущность в отдельном листе таблицы
- Лист "Инфо" содержит статистику и время последнего обновления
